To start sweep/faithfulness analysis/masking analysis, run any of the run_* python files and specify which model to use with --config_name=finetuned or --config_name=non_finetuned afterwards,
additional parameters can be specified by overriding the yaml parameters (for example sweep.queries_per_class=2 or lrp.mode=proto_margin). For the lrp-mode-specific params, it is important to override the values that were set in the configs/experiment/finetuned.yaml or config/experiemt/non_finetuned.yaml by specifying the lrp_params or sweep.params keys, as they get merged into the standart lrp/sweep keys, depending on what lrp.mode was set.
